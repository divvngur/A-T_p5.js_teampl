핵심 기술 

ml5.js (FaceMesh), Firebase, Canvas API 

 

게임 한 줄 소개 

ml5.js FaceMesh로 얼굴을 인식하여 입을 벌려 비둘기가 떨어뜨리는 알을 받고, 똥을 피하며 점수를 쌓는 카메라 인터랙션 기반 웹 게임 

 

1. 문서 개요 

▶  1.1 프로젝트 목적 

카메라 인터랙션(얼굴 인식)을 활용한 새로운 방식의 웹 게임 경험 제공 

ml5.js FaceMesh 기반 입 움직임 감지로 직관적이고 재미있는 플레이어 조작 구현 

Firebase 백엔드 연동으로 실시간 경쟁 순위표 및 사용자 기록 관리 

 

▶  1.2 기술 스택 요약 

구분 

기술 / 라이브러리 

프론트엔드 

HTML5, CSS3, JavaScript (Canvas API) 

AI / CV 

ml5.js (FaceMesh) — 얼굴 랜드마크 감지 

백엔드 / DB 

Firebase Authentication, Firestore 

디자인 툴 

Figma (프로토타입 → 프론트엔드 변환) 

 

▶  1.3 게임 모드 개요 

모드 

설명

생명 소진까지 무한 진행. 보스 등장. 결과 순위표에 기록.

게임의 첫 진입점이자 모드 선택의 허브 화면입니다.

7. 피그마 프로토타입 제작 가이드 

본 PRD를 기반으로 Figma AI 플러그인을 통해 프론트엔드 코드로 변환 시 권장사항 

 

▶  7.1 컴포넌트 기준 

모든 버튼은 Auto Layout 적용 (패딩: 수직 12px, 수평 24px) 

공통 색상 및 타이포그래피를 Figma Style / Variables로 등록 

게임 화면 오브젝트(비둘기, 알, 똥)는 별도 Component Set으로 구성 

팝업 / 모달은 Overlay 프레임으로 처리 

 

▶  7.2 화면 전환 프로토타이핑 

Smart Animate 트랜지션 활용 (화면 전환 300ms 권장) 

경고 팝업은 Present 오버레이로 구현 

카메라 가이드 화면은 인터랙티브 컴포넌트로 '진입 전 / 진입 후' 상태 구분 

 

▶  7.3 핸드오프 권장사항 

Figma Dev Mode 또는 AI 플러그인으로 CSS / React 컴포넌트 코드 추출 

게임 Canvas 영역은 별도 개발 구현 (Figma에서는 레이아웃 가이드만 정의) 

ml5.js 연동 영역은 카메라 뷰포트 크기 및 가이드 선 좌표를 Figma에서 측정 후 코드에 반영
